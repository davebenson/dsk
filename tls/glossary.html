• AES: Advanced Encryption Standard. A standardized block cipher. Invented by Joan Daemen and Vincent Rijmen. NIST accepted 128-, 192- and 256-bit variants. For our purposes, FIPS 197 specifies this algorithm.
  • AEAD, Authenticated Encryption with Associated Data: a means to convert a block cipher into one that can accept variable length plaintext and perform authentication. (It also can provide authenticated but unencrypted data, called "associated data" which is needed for ...) (GCM and CCM are the AEAD required by TLS 1.3)
  • Block cipher: a way of encrypting data of a fixed size using a key known to the encryptor and decryptor. They involve a routine pre-compute for a key ("Key Expansion"), then an optimized routine to do the block-decryption/encryption. These tend to do the heavy work of the encryption algorithm, but they need to be coupled with a means to check the authenticity and integrity of the message.
  • Ciphertext: the encrypted data
  • DES: a series of encryption standards developed by the US government that are now considered to be too weak, in part due to its 56-bit key, and slower modern algorithms. Essentially, it's been replaced by AES.
  • GCM "Galois/Counter Mode": mode of operation and AEAD that converts a block cipher into a variable-length cipher with authentication. It uses a finite-field with power-of-two order (finite fields are sometimes called Galois fields).
  • ECB, a classical mode of operation for a block cipher. It's trivial: simply break the input into blocks, encrypt or decrypt them, and concatenate the results. Cannot handle arbitrary length data, and is susceptible to replay and reordering attacks.
  • HMAC - Message Authentication Code
  • MAC - Message Authentication Code
  • MD5 - a venerable, but deprecated, cryptographic hash. Still widely used.
  • Nonce - A value that is used only once, often to avoid replay attacks.
  • Octet: a term used in RFC standards for "byte", an 8-bit value, usually represented as a number in the range 0 to 255.
  • Plaintext: the unencrypted data: either the input to the encryption algorithm, or, equivalently, the output of the decryption algorithm.
  • SHA1 - a venerable, cryptographic hash. Still widely used.
